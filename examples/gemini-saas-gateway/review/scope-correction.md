# Scope Correction

The Gemini response included implementation source code, package metadata, and build configuration.

These artifacts are outside the scope of sysande.

sysande is limited to:

- system analysis documentation,
- design decisions and assumptions,
- coverage planning,
- architecture and analysis diagrams,
- diagram source files,
- view reviews and consistency reports.

The useful parts of the Gemini response are therefore limited to the proposed coverage matrix and the LikeC4 and D2 diagrams. Even those remain unreviewed drafts because the required discovery and approval steps were skipped.

Programming languages, frameworks, packages, and executable gateway code must not be produced as sysande outputs. Technology names may be documented only when the user has already confirmed them as constraints.
