# sysande Scope

sysande is limited to system analysis and system design documentation.

## In scope

- discovery and requirement clarification,
- system boundaries and actors,
- assumptions and architecture decisions,
- coverage planning,
- structural, data, flow, behavior, and deployment views,
- diagram source files,
- generated diagrams,
- review and cross-view consistency documentation.

## Out of scope

- implementation source code,
- executable application scaffolding,
- framework or library selection,
- programming-language selection,
- package manifests and build configuration,
- infrastructure provisioning code,
- production-ready gateway, authentication, billing, or other implementations.

Technology names may appear only when they are already confirmed constraints of the system being documented. sysande must not introduce or recommend a technology stack by default.

The normal sysande output is documentation and diagrams.
